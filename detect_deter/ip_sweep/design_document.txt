垃圾回收机制，每两秒钟扫描一次ip地址扫描列表，将超过临界时间的表项删除。

ip地址扫描表项按hash值存储，这样扫描时，能快速定位当前系统是否已经有这个ip的表项。

hash=u32_ip%(255*4=1020)

临界时间为1-1000毫秒，默认为5毫秒。

CLI
开启某个域的ip地址扫描和临界时间
set zone zone_name ip-sweep [threshold time_value]
关闭某个域的ip地址扫描功能
no set zone zone_name ip-sweep
设置某个域的ip地址扫描的临界时间为默认值
set zone zone_name ip-sweep default


当域的ip地址端口扫描功能开启后，初始化一个hash头数组，大小为1020，当ip地址扫描功能关闭后，这个数组释放，根据入接口属于哪个域，找到这个域，判断此域的ip扫描是否开启，如果开启，则根据报文的源地址算出一个hash，根据hash值找到对应的链表头，然后比对链表里的值是否有何此ip一样的，有的话， 查看此ip的第一次匹配时间，如果超过了临界时间，则将第一次时间改为当前时间，匹配个数记为1.如果没超过临界时间，将匹配个数自增，如果>10时，则丢弃此报文，如果>10，且==11时，丢弃报文并报警。如果对比链表时，没有找到一样的ip时，在此链表后面加入此ip节点，计数记为1，时间记为当前时间。

在垃圾扫描时，如果当前时间-第一个报文的时间>临界时间，则认为此节点为垃圾，将其释放，回收内存。
